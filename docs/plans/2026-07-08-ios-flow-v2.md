## Deviations from the run-loop default (recorded up front)

- **No GitHub issue yet.** `gh issue create` was denied by the session's
  permission layer (external write). Rule 6 is satisfied by the design
  artifact URL above; an issue can be filed by Meng when the first PR opens.
- **PR timing.** Work is committed per-iteration on `feat/ios-flow-v2` but
  `gh pr create` (external write) is deferred until Meng gives the word.
  Each phase therefore terminates in verified-green HANDOFF state, not
  create-pr.
- **Iteration cap raised to 30** (default 5). Rationale: six phases, each
  needing act -> build/test -> simulator-QA sub-cycles. The effective guard
  is the no-progress rule (2 consecutive no-progress iterations -> stop and
  hand off), as in the flow7 loop (cap 40, finished in 9).
- **Visual fidelity is `HUMAN:`** -- judged by Meng from simulator
  screenshots against the v2.1 demo artifact.
